Diseño del Sistema: Arquitectura y Protocolos
El diseño lógico del sistema se fundamenta en el sistema operativo Linux como host principal, el cual actúa como el núcleo de confianza encargado de orquestar los demonios de red y los protocolos de seguridad en un entorno distribuido. Dentro de esta arquitectura, el servicio DNS (Bind9) cumple la función crítica de resolución de nombres, permitiendo que tanto los clientes como los servidores identifiquen la ubicación del Centro de Distribución de Llaves (KDC) dentro del dominio institucional fis.epn.ec. Paralelamente, el protocolo NTP (Chrony) se encarga de la sincronización de los relojes de sistema, un componente vital para la operatividad de Kerberos, ya que previene ataques de replicación al validar la vigencia temporal de cada ticket emitido.

La interacción entre los servicios se completa con la integración de LDAP y Kerberos, donde el primero funciona como el directorio centralizado de objetos y el segundo como el mecanismo de autenticación segura. LDAP provee la información descriptiva de los usuarios (nombres, roles, carreras) mientras que Kerberos gestiona la confianza mediante el intercambio de tickets de servicio para aplicaciones protegidas como el servidor web Apache. Esta arquitectura garantiza un flujo de datos coherente donde la resolución de nombres, la precisión temporal y la verificación de identidad convergen para ofrecer un entorno de Single Sign-On (SSO) eficiente y escalable para la FIS. Finalmente, para garantizar la persistencia en el entorno WSL2, se han aplicado configuraciones específicas en los archivos de montaje y red, asegurando la estabilidad del nombre de dominio totalmente calificado (FQDN).
